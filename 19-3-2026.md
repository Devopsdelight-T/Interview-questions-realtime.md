## 3. In Kubernetes, what is the difference between a liveness probe, a readiness probe, and a startup probe? When would you use each one, and what happens if you configure them wrong?
<details>
<summary>Click to expand answer</summary>

- **Liveness Probe**
  - Checks if app is alive
  - Failure → pod restart

- **Readiness Probe**
  - Checks if app is ready to serve traffic
  - Failure → removed from service

- **Startup Probe**
  - Used for slow-start apps
  - Prevents premature restarts

**Misconfiguration risks:**
- Too strict → unnecessary restarts
- Too lenient → bad pods serving traffic

</details>

## 4. You have a Kubernetes cluster where pods are running fine, but users are complaining about intermittent 502 errors. How would you troubleshoot this step by step?
<details>
<summary>Click to expand answer</summary>

Step-by-step:

1. Check **Ingress / Load Balancer**
2. Check **Service endpoints**
3. Check **Pod health**
4. Look at **application logs**
5. Check **timeouts / retries**
6. Inspect **resource usage**
7. Network issues (CNI)

**Common causes:**
- Pods failing readiness
- Timeouts
- Connection resets

</details>

## 10. What is the difference between Prometheus pull-based model and a push-based model like CloudWatch or Datadog? What are the pros and cons of each, and when would you choose one over the other?
<details>
<summary>Click to expand answer</summary>

**Prometheus (Pull):**
- Scrapes metrics
- Easy service discovery
- Better for dynamic systems

**Push systems (CloudWatch/Datadog):**
- Agents push data
- Works behind firewalls

**Trade-offs:**
- Pull = simpler, but needs access
- Push = flexible, but more overhead

</details>

## 12. What is the difference between a DaemonSet and a Deployment in Kubernetes?
<details>
<summary>Click to expand answer</summary>

- **Deployment**
  - Runs specified replicas

- **DaemonSet**
  - Runs one pod per node

**Use cases:**
- DaemonSet → logging, monitoring agents
- Deployment → applications

</details>
